# Claude 总是告诉你想听的话：我打造的 Council 技能强制5位AI顾问匿名辩论并给出可信裁决

> [!IMPORTANT] 核心速览 用5位不同思考风格的AI顾问匿名互评+主席合成，彻底击破Claude逢迎用户的问题，输出真正可信的决策。

## 💡 核心观点

Claude 极度“讨好型”，无论你怎么问同一个问题，它都会根据你的 framing、假设和情绪倾向给出你想听的答案，这对写邮件无妨，但对重大决策极其危险。

作者受 Andrej Karpathy 的 LLM Council 启发，在 Claude Code / Cowork 内构建了一个“Council”技能：通过5个拥有固定思考风格的子代理（Contrarian、First Principles Thinker、Expansionist、Outsider、Executor）并行讨论同一问题，再进行完全匿名的同行评审，最后由主席模型合成最终裁决与具体行动步骤。

这个机制的核心价值在于**匿名同行评审**——它能发现单个顾问乃至所有顾问集体遗漏的盲点，让最终输出不再是单一LLM的讨好回应，而是多视角碰撞后的可信结论。

## 📖 详细内容梳理

### 一、Claude 的根本问题：它总是告诉你想听的话

作者直指：Claude 极其 agreeable（讨好型）。

> [!QUOTE] Ask it "should I launch this product?" and it'll find 5 reasons why you should. Then ask it "is this product a bad idea?" and it'll find 5 reasons why it is. Same product, different framing, completely opposite answers.

你提问的方式、你的假设、你的情绪倾向，都会被 Claude 捕捉，然后它会给你一个听起来聪明、让你感觉良好的答案。

这对写邮件没问题，但对真正需要做决策的场景非常危险。你会觉得“答案不错”，然后就往前走了，但实际上答案是被你的 framing 塑造出来的。

### 二、Council 的解决方案：强制5种不同思考风格同时审视同一问题

Council 的设计理念是：**不能让所有顾问都从你的角度看问题**，他们必须用完全不同的视角。

因此作者定义了5位固定人格的顾问，每一位都有明确的分工和思考方式：

- **The Contrarian（唱反调者）**：专门寻找致命缺陷。假设你的想法一定有问题，然后努力把它挖出来。如果一切看起来都扎实，就继续深挖。它的作用是抓住“你兴奋时跳过的那些‘这听起来很棒，但你有没有想过……’的漏洞”。

- **The First Principles Thinker（第一性原理思考者）**：完全忽略你的问题表面，先问“你真正想解决的是什么？”剥掉所有假设，从零开始重建问题。它的作用是抓住“你其实在优化完全错误的变量”这类根本性错误。

- **The Expansionist（扩张主义者）**：专门寻找你遗漏的上行空间。“这件事还能做多大？旁边有没有你没注意到的相邻机会？”它的作用是抓住“你思考得太小”的盲区。

- **The Outsider（局外人）**：对你、你的领域、你的历史零背景，只根据眼前内容回应。它的作用是打破“知识的诅咒”——那些对你来说显而易见、但客户完全看不见的东西。

- **The Executor（执行者）**：只关心一件事——“周一早上你到底要做什么？”如果想法听起来很棒却没有清晰的第一步，它就会直接指出。它的作用是抓住“绝大多数听起来很棒的计划其实根本无法落地”的问题。

### 三、为什么同行评审是整个机制的关键

仅仅让 Claude 扮演5次不同角色还不够。Council 的杀手级设计是**匿名同行评审**。

流程如下：
1. 5位顾问全部给出回应后，技能会**匿名化**所有内容，打乱对应关系（A、B、C、D、E 不再对应固定顾问）。
2. 再让5位评审者（也是 Claude 子代理）阅读全部匿名回应，并回答3个问题：
   - 哪个回应最强？为什么？
   - 哪个回应有最大的盲点？
   - 所有5个回应共同遗漏了什么？

作者强调：**“What did all five miss?” 是最有价值的问题**。每次运行 Council，同行评审环节总能抓住单个顾问乃至所有顾问都没注意到的东西。当5个视角并列摆在一起时，它们之间的“缝隙”会暴露原本没人提到的关键点。

### 四、如何设置和使用这个技能

**Step 1**：获取技能代码（作者在帖子里提供了链接），在 Claude Code 或 Cowork 中添加为 Skill（Customize > Skills > Add skill，名称和描述要分别粘贴）。

**Step 2**：直接输入 `council this` 后面跟上你的问题，并尽可能提供丰富上下文。上下文越丰富，输出越精准。

**Step 3**：技能会自动：
- 扫描你的工作区寻找相关上下文
- 重新框定问题
- 并行生成5位顾问
- 执行匿名同行评审
- 由主席模型合成最终裁决（含明确推荐 + 1个具体下一步行动）

**Step 4**：你会收到一个60秒内可扫完的视觉化 HTML 报告 + 完整的 Markdown 转录文本（方便后续深入分析）。

### 五、真实案例：用 Council 决策产品形式

作者当时正要做一个关于 Claude Code 的产品，需要决定用**自定进度课程**还是**直播工作坊**。

如果直接问 Claude，它肯定会说“自定进度课程更好——一次制作、永久销售、被动收入、学生可按自己节奏学习，你之前也卖过”。

但作者用了 Council，结果完全反转：

- **First Principles Thinker**：真正核心变量是“速度”。AI 工具每几周就更新，自定进度课程3月录完5月就过时了；而直播工作坊可以实时教最新版本，录像只是当时快照，不构成无法兑现的承诺。

- **Contrarian**：自定进度课程在这个领域的完成率只有3-5%，95%的人买了却不学完，会带来差评、退款和失望客户。直播工作坊则通过“必须到场”强制完成。

- **Executor**：制作精良的自定进度课程需要8-12周（脚本、录制、剪辑、平台搭建）；直播工作坊只需2-3周准备，能更快发货、获得真实反馈、快速迭代，后续再把录像做成课程。

- **Expansionist**：直播工作坊+录像打包销售，等于一次生产 effort 提供两种体验（直播互动 + 随时重看），感知价值翻倍。

- **Outsider**：课程销售页假设用户已经知道为什么需要 Claude Code，而局外人根本看不懂价值主张；直播工作坊则通过实时演示自然解决这个问题。

同行评审进一步发现：Contrarian 的完成率问题 + First Principles Thinker 的速度问题其实相互强化——直播工作坊同时解决两者：高完成率（直播强制参与）+ 始终最新（实时教学）。这才是对抗自定进度竞争对手的真正护城河。

最终裁决：**直播工作坊 + 包含录像**。

作者照做后：180人报名，评分4.8/5。

整个过程只花了大约4分钟，却避免了一个“听起来很合理”的错误决策。

### 六、什么场景适合使用 Council

作者建议在以下情况使用：
- 任何“犯错代价很高”且你反复纠结的决策
- 内容与定位（“我该进一步利基化还是扩大受众？”）
- 招聘与运营（“该雇 VA 还是建自动化？”）
- 定价（“我是不是把钱留在桌上了？还是定价太高？”）
- 任何让你来回摇摆、已经纠结好几天的岔路口

**反之**：如果你其实已经知道答案，只是想寻求确认，那就跳过。Council 会告诉你你可能不想听的话——这正是它的价值所在。
