# Claude全家桶7天上手指南：不会编程也能建网站

> [!IMPORTANT] 普通人7天掌握Claude全家桶，从聊天到Vibecoding建站

**📅 发布时间：** 2026年3月30日 **👤 作者：** KK.aWSB（@KKaWSB） **🔗 来源平台：** X (Twitter)

## 💡 核心观点

作者作为不会写代码、也不打算学的普通人，亲身验证2026年Claude全家桶（Chat、Project、Cowork、Code）让“不会编程”彻底不再是限制：通过7天分阶段实践，从基础聊天到Project固化上下文、Cowork实时协作办公，再到Code一键建网站、自动化工具，普通人只需用中文清晰描述需求即可实现专业产出。  
核心在于认知升级：AI时代最重要的不是学编程，而是学会把模糊想法变成具体指令；Claude不是工具，而是你的千军万马，掌握它就能用AI构建想要的生活。  
作者强调这套方法适合所有非开发者，诚实指出烧额度、代码审查、死循环、审美局限等短板，但整体体验已让“不会编程”变成优势而非劣势。

## 📖 详细内容梳理

### 第0步：先搞清楚Claude全家桶是什么

很多人只知道“Claude”，其实它是一整套产品线。  
作者用一句话清晰解释每个产品：  
- **Chat**：就是聊天，跟ChatGPT一样，一问一答的对话机器人。  
- **Project**：还是聊天，但可以拆成独立的项目空间，每个项目有独立的上下文和指令。  
- **Cowork**：想象一下Google Drive和Claude Project生了个孩子。你可以在里面直接编辑文档、管理文件，同时让AI帮你干活。  
- **Code**：这是给开发者用的编程革命。但关键是——现在不会编程的人也能用它来建网站、做工具、写自动化脚本。  

四个产品的关系不是互相替代，而是层层递进：Chat是入门，Project是进阶，Cowork是日常主力，Code是终极武器。  
你不需要全部精通，但你需要知道每个工具适合什么场景。

> [!QUOTE] 这四个产品的关系不是互相替代，而是层层递进。

### Day 1-2：Chat和Project——把基本功打扎实

前两天最简单。  
如果你已经在用Claude聊天了，这部分可以快速过，但作者还是建议花时间检查几件事，因为很多人用了半年，基本功还是歪的。  

**第一件事：确保你在用对模型**  
Claude现在有好几个模型可以选。Opus 4.6是最聪明的，适合需要深度思考的复杂任务；Sonnet 4.6速度更快，日常对话和写作完全够用。  
别什么问题都用Opus，也别什么任务都用Sonnet。选对模型，效率差好几倍。  

**第二件事：学会用Project**  
Project本质上就是“带上下文的Claude”。  
你可以给一个Project设定系统指令（比如“你是一个财经分析师，风格简洁犀利”），上传参考文档（比如你的品牌手册、过往文章），然后在这个空间里的所有对话，Claude都会带着这些背景知识来回答。  

**举例**：作者有一个专门写推文的Project，里面放了写作风格指南、过往爆款文章案例、以及一套选题方法论。每次对话，它出来的东西就直接带着作者的调性，不需要每次重新解释。  
这就是Project的价值：把你的上下文固化下来，让每一次对话都不从零开始。  

**Day 1-2 的作业**：创建3个Project。一个工作用的，一个写作用的，一个学习用的。每个Project里，上传至少一份参考文档，写好系统指令。

### Day 3-4：Cowork——你的AI协作工作台

从第三天开始，事情变得有意思了。  
Cowork是Claude产品线里作者认为最被低估的一个。  
它长什么样？左边是Claude的对话窗口，右边是一个可以实时编辑的文档区域。你跟Claude聊天的同时，它可以直接在右边帮你写文档、改文档、整理文档。  

**跟Chat的区别**：  
Chat是你跟Claude一来一回地聊，最后自己复制粘贴结果。  
Cowork是你跟Claude一起在同一个工作台上干活，文件就在那里，随时改、随时看。  

**实际场景举例**：  
- **场景一：写长文章**。作者不在Chat里写长文章，因为复制粘贴太低效。在Cowork里跟Claude说“帮我写一篇关于XX的文章”，它直接在右边文档区域里写。看到不满意的段落，直接框选说“这段改一下，太书面了”，它当场改。整个过程就像坐在AI写手旁边，指着屏幕说“这里改改那里调调”。  
- **场景二：整理资料**。把一堆杂乱的笔记、会议记录、参考链接丢进Cowork，让Claude帮你整理成结构化的文档。它会自动归类、提炼要点、生成摘要。  
- **场景三：协作工作流**。在Cowork里放进品牌指南、过往作品、团队规范，然后让Claude基于这些素材执行任务。它会先读文件，再按你的风格产出内容。  

**Day 3-4 的作业**：把你手头一个真实的工作任务搬进Cowork完成。不是练习，是真实任务。写一份报告、整理一份方案、做一份周报，随便什么都行。用真实任务来体验，你才会真正理解Cowork跟Chat的区别有多大。

### Day 5-7：Code——不会编程也能建网站

重头戏来了。  
前四天掌握了聊天、项目管理和文档协作能力。现在要做一件听起来不可能的事：让不会写代码的你，用Claude Code来建一个真正的网站。  

**关系大了**：  
Claude Code能做的事情远不止写代码：一句话做一个公司官网、把Excel变成可视化数据看板、做一个互动式产品展示页面、生成客户调查问卷并自动收集数据……  
所有这些，你不需要写一行代码，只需用中文告诉它你想要什么。  
英语是新时代的编程语言，中文也是。  

**怎么设置Claude Code（不需要任何代码知识）**：  
1. **创建一个免费的GitHub账号**：github.com注册，代码的“云端仓库”，像代码版的百度网盘。  
2. **在Claude桌面端连接GitHub**：设置 → 连接器（Connectors） → 搜索GitHub授权。  
3. **打开Claude Code，开始“说话建站”**：桌面应用顶部点Code标签，选择Opus 4.6模型，打开“自动接受编辑”。  
   输入模板提示词：  
   “创建一个GitHub仓库，命名为'my-website'。我不会写代码，也不想学。帮我完成所有编码工作。尽量不要请求权限确认。我要做的事：为我的个人品牌做一个简洁的展示网站。包含：首页大标题、个人介绍、服务列表、联系方式。风格参考：[附上你喜欢的网站截图]”  
   然后看着它自动写代码、创建文件、搭建页面。  
4. **看效果，反复修改**：生成后让Claude给在线访问链接。把不对的地方记下来，回到Code里说“标题太小了，放大一倍。背景换成浅灰色……”Claude一条条改，你刷新页面看最新效果。  

这个“说→看→改”的循环，就是Vibecoding（氛围编程）。你不写代码，只描述想要的结果，AI帮你实现。  

**让速度再快100倍：VS Code + 跳过权限**：  
用一段时间会发现每次编辑文件、创建文件、运行命令都弹窗问“允许吗？”一个小网站可能要点20多次。  
解决方法：下载VS Code，安装Anthropic官方Claude Code扩展，登录账号，在设置里打开“Skip Permissions”。从此Claude不再每个操作都问，你回来看结果就行。  

**给你的项目一个“记忆”**：  
Claude Code默认每次打开从零开始。解决办法：在第一次完成项目后，跟Claude说：“在项目根目录创建一个CLAUDE.md文件。把你目前了解的关于这个项目的所有信息写进去：文件夹结构、每个文件的作用、我的设计选择（字体、颜色、布局）、我的偏好、已有的页面和模块。”  
Claude会生成记忆文件。下次打开，它先读文件，自动恢复所有上下文。你说“加一个博客页面”，它就会用跟你之前一致的风格来建——不需要重新解释。  

### 几句诚实的话（短板与注意事项）

Claude Code不是万能的，作者诚实指出短板：  
- **烧额度快**：一次Code会话相当于二三十次普通Chat。如果你每天重度使用，20美元/月的Pro计划可能一周就到顶，建议上100美元/月的Max计划。  
- **没法审查代码质量**：你看不懂代码。如果它写错，我们大概率发现不了。解决办法：用眼睛测——打开网站，点每一个按钮，在手机上看效果。  
- **有时候陷入死循环**：遇到bug修了又产生新bug。解决办法：看到同一个错误出现两次，直接打断说“停。解释一下哪里出了问题。给我两种不同的解决方案。”  
- **默认设计审美一般**：能用，但像模板。你需要主动推它——给截图、指定字体、配色、间距。如果你只说“做好看点”，出来的东西就是“不丑但也不出挑”。参考图是你最好的武器。

### 一周之后的变化

七天时间，你会经历这样一个变化：  
- Day 1-2：学会用Project把上下文固化下来，不再每次对话从零开始。  
- Day 3-4：学会用Cowork在AI旁边协作办公，效率翻倍。  
- Day 5-7：学会用Code建网站、做工具，不写一行代码。  

这不是一个技术学习的过程，而是一个认知升级的过程。  
你会意识到，“不会编程”这件事，在2026年已经不再是一个限制了。  
你不需要学会写代码，你需要学会的是：清晰地描述你想要什么。  
这才是AI时代最重要的技能。不是Prompt工程，不是编程语言，而是你能不能把脑子里模糊的想法，变成一句清晰的、具体的、可执行的指令。  
能做到这一点的人，AI就是他们的千军万马；做不到的人，AI就只是一个高级搜索引擎。差距不在工具，在思维方式。  
这一周，只是个起点。
