## 📖 详细内容梳理

### 1: HOW AGENTS WORK（Agent工作原理）

理解Agent工作原理非常重要，否则你根本不知道自己是否需要一个Agent。  
所有Agent的核心循环完全一致：  
**用户输入 → LLM思考 → LLM决策（回复或调用工具）→ 如果调用工具则执行并将结果反馈回去 → 重复循环**  

LLM是“大脑”（负责推理），工具是“手”（执行具体动作，如计算器、网页搜索、文件读写），记忆是“记事本”（记录已发生的事）。  
无论你用LangGraph、CrewAI、Anthropic SDK还是OpenAI Agents SDK，框架只是对这个循环的封装，本质不变。  

**Augmented LLMs（增强型LLM）**  
普通LLM只接受文本、输出文本。增强型LLM额外具备三大能力：  
- **Tools**：模型可调用的函数（Anthropic用input_schema，OpenAI用function object）。  
- **Retrieval**：从外部来源拉取信息（搜索引擎、文档、向量数据库）。  
- **Memory**：跨交互保留信息（消息历史或持久存储）。  

**Workflows vs. true agents（工作流 vs 真正Agent）**  
- **Workflows**：确定性，由代码控制执行路径，输入相同则输出路径相同。适合固定步骤的任务，更便宜（LLM调用少）。  
- **Agents**：动态，由LLM决定下一步，可能反复调用工具。适合开放式任务，但成本更高。  
建议：先用简单工作流尝试，如果不够再升级为自主Agent。

> [!QUOTE] The LLM is the “brain” that reasons. Tools are the “hands” that perform actions… Memory is the “notepad”…

### 2: THE FIVE CORE WORKFLOW PATTERNS（五大核心工作流模式）

大多数问题其实不需要完全自主的Agent就能解决。Anthropic总结并被广泛采用的5种模式，几乎覆盖了所有常见场景。每种模式都依赖增强型LLM。

**Pattern 1: Prompt chaining（提示链）**  
将任务拆成顺序步骤，每个LLM调用处理上一步输出，并在步骤间加入程序化“门控”验证质量。  
适用：可清晰拆解成固定子任务的任务。  
示例：生成营销文案 → 翻译；写大纲 → 验证 → 写全文。

**Pattern 2: Routing（路由）**  
对输入进行分类，然后路由到专属处理器，每个处理器有优化后的prompt。  
适用：不同类别输入需要完全不同处理方式（如客服分流）。

**Pattern 3: Parallelisation（并行化）**  
同时运行多个LLM调用。Sectioning：将任务拆成独立子任务并行；Voting：同一任务多次运行后聚合结果提升置信度。  
适用：子任务相互独立或需要共识的关键决策。

**Pattern 4: Orchestrator-workers（编排者-工作者）**  
中心LLM（编排者）动态拆解任务并委托给工作者LLM。子任务在运行时决定，而非预定义。  
适用：无法提前预测结构的复杂任务（如跨文件代码生成、研究、报告撰写）。

**Pattern 5: Evaluator-optimiser（评估者-优化者）**  
一个LLM生成输出，另一个评估并反馈；不通过则循环优化，直到满足质量标准。  
适用：有明确评估标准且迭代能显著提升质量的任务（如翻译、代码生成、写作）。

### 3: BUILDING YOUR AGENT（构建你的Agent）

这是大家最关心的部分。  
把“我想要一个做XYZ的Agent”变成现实的最简单思路：  
1. 写下工作内容  
2. 决定需要哪些工具  
3. 告诉模型如何表现  
4. 用5个真实案例测试  
5. 只有失败时才增加复杂度  

不需要掌握5个框架。推荐起步选择：  
- **Anthropic**：适合需要工具、文件、shell命令、网页操作、编码工作流的“能干助手”。  
- **OpenAI**：适合干净的开发者SDK、托管工具、handoffs、guardrails、生产路径。  

**最简单的思维模型**  
构建Agent前先回答4个问题：  
1. **预期产出是什么？**（e.g. 研究课题并写总结）  
2. **需要什么信息？**（网页搜索、文件、数据库等）  
3. **允许执行什么动作？**（仅回复、搜索、编辑文件、发邮件、写代码等）  
4. **必须遵守什么规则？**（语气、格式、约束、安全、什么算“好”）  

回答清楚这4个问题，通常一天就能做出第一个版本。

**初学者友好公式**  
Agent = 角色 + 目标 + 工具 + 规则 + 输出格式  
示例：  
角色：加密项目研究助理  
目标：找到准确信息并清晰总结  
工具：网页搜索、文件搜索、计算器  
规则：引用来源、不猜测、标明不确定  
输出格式：总结、风险、机会、最终判断  

**5种适合新手的入门Agent类型**（不要一开始就做多Agent群）：  
1. **Research agent**：收集信息并总结（需网页/文件搜索）。  
2. **Content agent**：写作、改写、总结（强system prompt + 可选文件访问）。  
3. **Workflow agent**：执行可重复业务流程（如分类支持票据）。  
4. **Personal knowledge agent**：基于你的文档回答问题（文件搜索/RAG）。  
5. **Operator agent**：在环境中执行动作（如读写文件、运行shell）。

**自定义Agent的实用清单**（避免新手常见错误）：  
- 工作要窄而具体  
- 定义明确输出格式  
- 给出示例  
- 只在必要时添加工具  
- 用真实（凌乱）的用户prompt测试  

**构建路径5步法** + **最大错误警示**（不要一开始就做“全能超级Agent”，先做最小可用版本）。

### 4: UTILISING TOOLS（工具使用）

大多数人把工具想反了：更多工具 ≠ 更聪明Agent。  
**更好的工具 + 更少的工具 = 更可靠Agent**。  

工具本质是“模型自己做不到的事”（计算、搜索、读文件、发邮件、查数据库）。  
**5步实用指南**：  
1. 先问“是否真的需要工具？”（能纯推理解决就别加）  
2. 用AI帮你设计工具（给出具体prompt模板）  
3. 保持简单（一个工具一个明确职责）  
4. 明确告诉Agent**什么时候**使用工具  
5. 允许Agent失败并修复（真实测试 + 规则优化）  

### 5: GIVE YOUR AGENT MEMORY（赋予记忆）

大多数人把记忆想得太复杂。  
其实只有两种记忆：  
1. **短期记忆**（对话历史，默认就有）  
2. **长期记忆**（外部知识，如笔记、PDF、数据库）  

**何时真正需要记忆？**  
- 需要跨消息记住 → 短期  
- 需要使用外部文档 → 长期  
否则大概率不需要。  

**3种选择**：  
A. 无记忆（70%场景够用，从这里开始）  
B. 对话记忆（SDK默认处理）  
C. 文件-based记忆（简单RAG）  

**关键规则**：Agent如果不用记忆就能正常工作，就不要加。

### 6: MAKING YOUR AGENT WORK IRL（让Agent真正落地）

Agent要么很烂，要么很强，区别在于：  
**坏prompt、无测试、不切实际的预期**。  

**5步落地指南**：  
1. 用AI生成真实测试案例  
2. 用真实用户风格（凌乱prompt）测试  
3. 一次只修复一个问题  
4. 用AI帮你调试Agent  
5. 不要过早复杂化（多Agent、复杂工作流等要等单Agent稳定后再加）

### 7: MULTIPLE AGENTS（多Agent协作）

**不要过早进入多Agent**。  
大多数人不需要。  
只有在以下3种情况才考虑：  
1. 技能明显不同  
2. 有清晰流水线  
3. 需要不同权限  

**安全模式**：Supervisor模式（用户 → 主Agent → 按需调用其他Agent）。  
**角色要简单**，慢慢添加（先1个，再最多2个）。

### 8: WRAPPING THIS ARTICLE UP!（总结）

Agent概念简单（核心循环50行Python就能实现），但操作上要求很高。  
真正工作在工具设计、错误处理、评估和“何时用简单工作流就够”的判断力上。  

**3个可立即行动的 takeaway**：  
1. 先从零手写一个Agent，理解底层循环。  
2. 从能解决问题的最简单模式开始，只有必要时才升级为自主Agent。  
3. 尽早投入工具设计和评估（好工具 + 20个真实测试案例 > 换模型或框架）。
