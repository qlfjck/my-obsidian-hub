### 如何使用

#### 下载安装
- 打开官网 https://obsidian.md/
- 根据自己的系统下载对应的版本

#### 新建笔记仓库
- 在下载过程中打开GitHub（没有账号的先注册），点击New Repository创建仓库
- 创建完成后复制仓库链接
- 在本地找一个文件夹，执行命令：
  ```
  git clone 链接地址
  ```
- 添加一个 `.gitignore` 文件，内容如下（防止不想上传的文件进入仓库）：
  ```
  .obsidian/workspace.json
  .obsidian/workspace-mobile.json
  ```

#### 打开本地仓库
- 下载安装完成后，打开刚刚clone下来的文件夹

#### 配置Git同步
- 点击第三方插件 → 先将安全模式关闭
- 点击浏览，在搜索框输入“git”，下载第一个插件（Git插件）
- 在选项中注意配置更新时间，并打开下方标识的按钮
- 修改后右侧会出现黑框，提示pull操作

#### 图片插件
- 默认图片渲染不符合Markdown语法且放置随意，需安装Custom Attachment插件
- 在插件市场搜索“Custom attachment”安装并启动
- 点击选项配置以下内容：
  - Markdown URL 格式：`assets/${noteFileName}/${generatedAttachmentFileName}`
  - 按照作者图片配置全部勾选一致
- 点击“文件与链接”修改下面两个选项
- 将图片复制进来即可看到路径，通过修改`[]`内的数字改变图片尺寸

至此基础配置完成，接下来学习Markdown语法。

### Markdown语法基础

Markdown是一种轻量级标记语言，在Obsidian中用于排版笔记内容。

#### 标题分级
使用`#`符号创建不同级别标题，`#`数量决定级别：
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

#### 文本格式化
- 加粗文本：`**这是加粗文字**`
- 斜体文本：`*这是斜体文字*`
- 高亮文本：`==这是高亮文字==`
- 删除线：`~~这是删除线文字~~`

#### 列表与任务管理
- 无序列表：`-`或`*`开头
  ```
  - 列表项1
  - 列表项2
    - 子列表项
  ```
- 有序列表：数字+点号
  ```
  1. 第一项
  2. 第二项
  ```
- 任务列表：`- [ ]`
  ```
  - [x] 已完成的任务
  - [ ] 未完成的任务
  ```

#### 代码块
- 行内代码：`` `代码示例` ``
- 代码块：三个反引号包裹，可指定语言
  ```
  ```python
  def hello_world():
      print("Hello, World!")
  ```
  ```

#### 表格
使用竖线和横线创建：
```
| 列标题1 | 列标题2 | 列标题3 |
|---------|---------|---------|
| 内容1   | 内容2   | 内容3   |
| 内容4   | 内容5   | 内容6   |
```

#### 链接与图片
- 链接：`[显示文字](https://example.com)`
- 图片：`![图片描述](图片链接)`

### AI结合Obsidian使用方式

Obsidian的纯文本架构天然适配AI工具，可极大提升笔记管理效率。

#### AI自动化笔记管理
通过AI工具（如Gemini CLI）实现：
- 自动创建笔记：AI根据指令自动生成笔记文件和文件夹结构
- 生成内容大纲：输入主题，AI自动生成详细文章大纲和框架
- 批量整理笔记：AI批量处理零散内容，整理为结构化笔记

示例操作：
> [!QUOTE] # 使用claude code 生成笔记  
> claude "帮我基于晴天写一首诗放到随笔里面"  
> AI会自动创建文件,然后写入到随笔里。

#### Markdown文件批处理
AI可执行以下批量操作：
- 重命名笔记时自动更新所有附件路径
- 将零散字幕或OCR内容整理为连贯段落
- 自动添加YAML元数据（标题、日期、标签等）
- 批量格式化代码块和表格

#### 智能内容生成
- 内容摘要：指令“总结这篇文章的核心要点，用列表形式呈现” → 输出带复选框的Markdown段落
- 智能关联：AI分析笔记内容，自动建立双向链接；在关系图谱中智能标注相关主题；推荐相关笔记和参考资料

#### 常用AI工具集成
- **Gemini CLI**：执行自然语言指令处理笔记，支持批量操作和自动化脚本，输出标准Markdown格式
- **Claude Code**：分析代码片段并插入注释，通过代码块与AI交互，生成技术文档和API说明

集成方式：
- 安装对应的AI工具CLI版本
- 在Obsidian中通过命令行或插件调用
- 设置自动化工作流，如每日自动总结笔记

### 双向链接与知识图谱

双向链接是Obsidian最核心的特性之一，让笔记之间建立有机联系，构建个人知识网络。

#### 什么是双向链接
- 在笔记A中引用笔记B时，笔记B会自动显示被笔记A引用
- 形成双向关联关系，让知识点自然串联而非孤立

#### 如何使用双向链接
- 基本语法：`[[笔记名称]]`
- 引用文件示例：`我本月写了一个教程 [[Obsidian零基础教程]]`
- 链接到标题：`[[笔记名称#标题名称]]` 如 `[[Obsidian教程#安装步骤]]`
- 自定义显示文字：`[[实际笔记名称|显示的文字]]`
- 嵌入笔记内容：`![[笔记名称]]`（在当前笔记显示被引用内容）

#### 创建双向链接的方法
1. **拖拽创建**：编辑模式下从左侧文件列表拖拽笔记到编辑区，自动生成链接
2. **输入创建**：输入`[[`后Obsidian自动提示现有笔记；笔记不存在时点击自动创建
3. **反向链接面板**：每篇笔记右侧显示“反向链接”面板，列出所有引用当前笔记的其他笔记，点击快速跳转

#### 标签系统
- 创建标签：`#标签名称` 如 `#技术/编程/Python`
- 作用：快速分类检索、在图谱中不同颜色显示、支持嵌套标签（用`/`分隔）

#### 知识图谱
- 打开方式：左侧边栏“关系图谱”图标 或 快捷键 `Ctrl/Cmd + G`
- 功能：
  - 节点：每篇笔记为一个节点
  - 连线：表示笔记间链接关系
  - 颜色：可根据标签或文件夹设置
  - 筛选：筛选特定标签或文件夹笔记
- 操作：拖拽节点调整布局、点击节点打开笔记、放大缩小查看整体结构

实用技巧：
> [!TIP] 在笔记中建立知识网络：[[核心概念]] → [[具体应用]] → [[实践案例]]  
> 使用标签分类：#项目管理 #个人成长 #技术学习

### 好用的插件推荐

Obsidian强大之处在于丰富插件生态，分为核心插件（官方自带）和社区插件（第三方）。

#### 必备核心插件
1. **大纲（Outline）**：实时显示当前笔记标题结构，点击快速跳转
2. **工作台（Workspaces）**：保存不同工作布局，一键切换场景
3. **录音机（Audio Recorder）**：笔记内直接录音，文件自动保存在库
4. **模板（Templates）**：创建笔记模板，避免重复输入；支持动态变量；设置：创建Templates文件夹 → 设置中指定位置 → 使用`{{date}}`等变量 → 插入模板

#### 热门社区插件
1. **Calendar**：侧边栏日历视图，点击日期创建/打开日记
2. **Advanced Tables**：快速创建编辑Markdown表格，支持排序和公式
3. **Templater**：比核心模板更强大，支持JavaScript脚本和动态模板
   示例：
   ```
   ---
   title: <% tp.file.title %>
   date: <% tp.file.creation_date("YYYY-MM-DD HH:mm:ss") %>
   tags: 
   ---
   ```
4. **Dataview**：将笔记当作数据库查询，生成动态列表
   示例查询：
   ```
   TASK
   WHERE !completed
   ```
5. **Excalidraw**：在Obsidian中绘制手绘风格图表（流程图、思维导图），可嵌入笔记
6. **Auto Link Title**：粘贴URL时自动获取网页标题，生成格式化链接

#### 插件使用建议
- 循序渐进：不要一次性安装太多，先熟悉基本功能
- 按需安装：根据实际场景选择
- 定期清理：卸载不常用插件，保持流畅
- 查看文档：每个插件都有说明文档，先查阅
- 社区交流：加入Obsidian中文社区获取推荐和技巧

#### 插件配置技巧
- 设置快捷键：设置 → 快捷键 → 搜索插件命令 → 自定义快捷键
