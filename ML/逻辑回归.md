# 逻辑回归完整笔记

> 涵盖：模型本质 · 与线性回归的关系 · Sigmoid · 损失函数 · 交叉熵推导 · 凸性证明 · 梯度下降训练

---

## 1. 逻辑回归做的一件事

**把输入映射成"属于某一类的概率"。**

它回答的问题形式永远是：这封邮件是垃圾邮件的概率是多少？最终用概率来做分类。

### 三步流程

**第一步：线性计算**

把所有特征加权求和，得到一个任意实数 z：

```
z = w₁x₁ + w₂x₂ + ... + b
```

**第二步：Sigmoid 压缩**

用 Sigmoid 函数把 z 压进 [0, 1]，变成概率：

```
P = σ(z) = 1 / (1 + e^(-z))
```

- z 越大 → P 越接近 1
- z 越小 → P 越接近 0
- z = 0 → P = 0.5

**第三步：用 0.5 做决策**

```
P ≥ 0.5  →  预测为 1（正类）
P < 0.5  →  预测为 0（负类）
```

---

## 2. 为什么不直接用线性回归做分类

### 线性回归做分类的思路

把标签 0/1 当数字，拟合一条直线 `y = wx + b`，输出 ≥ 0.5 就判为类别 1。

### 三个硬伤

| 问题 | 说明 |
|------|------|
| 输出越界 | 线性回归输出可以是 1.7 或 -0.3，无法解释为概率 |
| 极端值干扰 | 一个远处样本会把整条直线拉偏，决策边界漂移 |
| 损失函数不配套 | MSE 在分类任务里是非凸的，梯度下降容易卡住 |

---

## 3. 为什么必须套 Sigmoid

Sigmoid 不是随意选的"壳"，它做了两件有意义的事：

1. **把任意实数压进 (0, 1)**，让输出有概率的含义
2. **对极端值不敏感**——曲线两端饱和，中间灵敏，边界稳定

从数学角度：只要想用概率做二分类，从最大似然推导出来的激活函数就是 Sigmoid，不是拍脑袋选的。

---

## 4. 损失函数：为什么用交叉熵

### 损失函数是什么

损失函数是一个**打分员**，回答一个问题：

> 模型猜了个答案，猜得有多离谱，该罚多少？

横轴 = 错了多少，纵轴 = 被罚多少。曲线形状决定训练时梯度的大小。

### 常用损失函数一览

#### 回归任务

| 名称 | 公式 | 特点 | 适用场景 |
|------|------|------|----------|
| MSE | `(y - ŷ)²` | 对离群点极敏感，平方放大惩罚 | 数据干净、无异常值 |
| MAE | `|y - ŷ|` | 对离群点鲁棒，但原点不可导 | 数据含异常值 |
| Huber | 小误差用 MSE，大误差用 MAE | 兼顾光滑与鲁棒，需调 δ | 少量异常值，想要平滑梯度 |

#### 分类任务

| 名称 | 公式 | 特点 | 适用场景 |
|------|------|------|----------|
| 二元交叉熵 | `-[y·log(P)+(1-y)·log(1-P)]` | 严格凸，训练稳定 | 二分类默认选择 |
| Focal Loss | `-α(1-P)^γ·log(P)` | 自动聚焦难样本 | 类别极度不平衡 |
| Hinge Loss | `max(0, 1-y·ŷ)` | 鼓励大间隔分类 | SVM |

#### 其他

| 名称 | 适用场景 |
|------|----------|
| KL 散度 | VAE、生成模型、知识蒸馏 |
| CTC Loss | 语音识别、OCR 等序列任务 |
| Triplet Loss | 人脸识别、相似度学习 |

### 为什么不用 MSE 做分类

MSE 套在 Sigmoid 外面，损失曲面变成**非凸**的——梯度下降可能卡在局部最小值，永远找不到最优解。

## 5. 交叉熵推导：从最大似然到公式

### 第一步：想清楚目标

训练模型 = 找一组参数 w，让"在这组参数下，观测到当前数据的概率"最大。这叫**最大似然估计**。

### 第二步：一个样本的"合理程度"

"合理程度"= 用当前参数，能把眼前这个数据解释得多好。

- 真实 y=1，模型预测 P=0.9 → 合理程度 = **0.9**（猜对了且有把握）
- 真实 y=0，模型预测 P=0.9 → 合理程度 = **1-0.9=0.1**（猜反了还很自信）

### 第三步：用一个式子合并两种情况

```
合理程度 = P^y · (1-P)^(1-y)
```

验证：
- y=1 时 → P¹·(1-P)⁰ = P ✓
- y=0 时 → P⁰·(1-P)¹ = 1-P ✓

这个式子的巧妙之处：用指数 y 和 (1-y) 作为开关，自动在两种情况间切换。

### 第四步：取对数

N 个样本的联合概率是连乘，数值极小会下溢。取 log 后变成求和，计算稳定：

```
log 合理程度 = y·log(P) + (1-y)·log(1-P)
```

为什么 log 的形状"正好对"：
- P 接近 1 时，log(P) 接近 0 → 损失小
- P 接近 0 时，log(P) → -∞ → 损失暴涨

### 第五步：最大化 → 最小化，加负号

习惯最小化损失，所以加负号：

```
L = -[y·log(P) + (1-y)·log(1-P)]
```

**这就是交叉熵。** 它是从"最大似然"出发，经过取 log、加负号两步推出来的必然结果，不是拍脑袋设计的。

## 6. 交叉熵的凸性证明

### 为什么凸性重要

- 凸函数 → 只有一个全局最小值 → 梯度下降必然收敛
- 非凸函数 → 可能有局部最小值 → 训练结果看运气

### 凸性来自三个事实的叠加

```
z = wx + b          ← 线性函数，二阶导 = 0（既凸又凹）
        ↓
P = σ(z)            ← 单调递增，保持凸性的传递
        ↓
-log(P)             ← 凸函数（对 P 二阶导 = 1/P² > 0）
        ↓
凸函数 ∘ 线性函数   ← 仍是凸函数（保凸性）
        ↓
N 个样本求平均      ← 凸函数的非负加权和仍是凸函数
```

结论：J(w) 关于参数 w 严格凸。

### 二阶导数验证（单样本，y=1）

```
J(w) = -log(σ(wx+b))

一阶导：dJ/dw = (P - y)·x

二阶导：d²J/dw² = P·(1-P)·x²
```

因为 P ∈ (0,1)，所以：
- `P·(1-P) > 0` 恒成立
- `x² ≥ 0` 恒成立
- 二阶导 ≥ 0，**严格凸得证**

### 对比：为什么 MSE + Sigmoid 非凸

MSE 损失：`J(w) = (σ(wx+b) - y)²`

对 w 求二阶导，会出现 `σ''(z) = P(1-P)(1-2P)` 这一项，当 P > 0.5 时为负，导致二阶导**可以变号**，凸性不能保证。

## 7. 梯度下降训练过程

### 直觉

梯度 = 当前站的地方，坡往哪边倾斜。每次往下坡方向走一小步，反复走，到达碗底（损失最小）。

```
w = w - lr × 梯度
```

其中 lr（learning rate，学习率）控制每步走多大。

### 完整流程

**第一步：随机初始化**
```python
w = 0.01
b = 0
```

**第二步：前向传播，算预测概率**
```python
z = w * x + b
P = 1 / (1 + np.exp(-z))
```

**第三步：计算损失**
```python
L = -np.mean(y * np.log(P) + (1 - y) * np.log(1 - P))
```

**第四步：计算梯度**
```python
# 交叉熵对 w 的梯度，化简后出奇地干净：
dw = np.mean((P - y) * x)
db = np.mean(P - y)
```

> 注意：梯度形式 `P - y` 和线性回归的 MSE 梯度长得一模一样。
> Sigmoid 和 log 在求导时互相抵消，结果异常干净。

**第五步：更新参数**
```python
w = w - lr * dw
b = b - lr * db
```

**第六步：重复迭代**

重复第二到第五步，直到损失不再明显下降。

### 完整代码

```python
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def logistic_regression(X, y, lr=0.1, epochs=1000):
    w = np.zeros(X.shape[1])
    b = 0

    for epoch in range(epochs):
        # 前向传播
        z = X @ w + b
        P = sigmoid(z)

        # 计算损失（交叉熵）
        loss = -np.mean(y * np.log(P + 1e-9) + (1 - y) * np.log(1 - P + 1e-9))

        # 计算梯度
        dw = (X.T @ (P - y)) / len(y)
        db = np.mean(P - y)

        # 更新参数
        w -= lr * dw
        b -= lr * db

        if epoch % 100 == 0:
            print(f"Epoch {epoch}, Loss: {loss:.4f}")

    return w, b

def predict(X, w, b, threshold=0.5):
    P = sigmoid(X @ w + b)
    return (P >= threshold).astype(int)
```

## 8. 核心结论汇总

| 问题 | 结论 |
|------|------|
| 逻辑回归做什么 | 线性计算 → Sigmoid 压缩 → 输出概率 → 分类 |
| 为什么不用线性回归做分类 | 输出越界、受极端值影响、MSE 非凸 |
| 为什么必须套 Sigmoid | 保证输出在 (0,1)，边界稳定，数学上是最大似然的必然结果 |
| 交叉熵从哪来 | 最大似然 → 取 log → 加负号 → 得到交叉熵 |
| 交叉熵为什么是凸的 | 凸函数复合线性函数保凸，二阶导 P(1-P)x² 恒 ≥ 0 |
| 梯度公式为什么这么简单 | Sigmoid 和 log 求导时互相抵消，化简为 (P-y)x |
